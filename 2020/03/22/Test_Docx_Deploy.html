Coming to the 2nd model, I need to ensure the that function related objects have a different name when compared to the 1st model. This is to avoid any name clashes.

For example, the first function that gets created in my model is UUGAEFR in both my models. The challenge here is that, if the same name exists but with different SCREEN ID’s, then the function that gets invoked is dependent on the first such object encountered in the interactive library list being used in Web Option.

If the library list for RXD87MDL_1 is set, then when I invoke “UUAGEFR”, “Edit Students” is invoked. If library list for RXD87MDL_2 is set, then “Edit Product_File” is invoked. This is because both the functions are named as UUAGEFR.

To overcome this, in 2nd model RXD87MDL_2, I have set the YOBJPFX model value to “VV” instead of “UU. This ensure that both my access paths and function names start with “VV” instead of “UU”.
